\# Design Specification



\## Objetivo



Criar uma experiência simples, rápida e agradável.



O aplicativo deve transmitir a sensação de um assistente inteligente, e não de um sistema de controle.



\---



\## Personalidade



\- Simples.

\- Moderna.

\- Organizada.

\- Confiável.

\- Inteligente.



\---



\## Diretrizes



\- Poucos toques para executar qualquer ação.

\- Fluxos curtos.

\- Pouca digitação.

\- Interface limpa.

\- Informações organizadas por prioridade.



\---



\## Navegação



Priorizar acesso rápido às funcionalidades principais.



O usuário deve conseguir registrar uma compra em poucos segundos.



\---



\## Fluxos Prioritários



\### Registrar compra



Fluxo principal.



Deve ser extremamente rápido.



\---



\### Lista de compras



Principal tela do sistema.



Sempre facilmente acessível.



\---



\### Inventário



Atualização rápida.



Não exigir muito tempo do usuário.



\---



\### Produtos



Apresentar apenas informações relevantes.



Evitar excesso de detalhes.



\---



\## IA



As sugestões da IA devem:



\- ser identificadas;

\- poder ser aceitas;

\- poder ser editadas;

\- poder ser rejeitadas.



Nunca devem parecer decisões definitivas.



\---



\## Comunicação



Utilizar linguagem simples.



Evitar termos técnicos.



Sempre explicar recomendações de forma objetiva.



\---



\## Mobile First



A experiência principal é em dispositivos móveis.



Desktop deve funcionar, mas não é prioridade.



\---



\## Liberdade Criativa



O agente responsável pelo design possui liberdade para definir:



\- estrutura das telas;

\- navegação;

\- componentes;

\- organização visual;



desde que respeite os princípios definidos neste documento.
